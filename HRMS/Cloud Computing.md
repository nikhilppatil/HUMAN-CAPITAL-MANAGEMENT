***1. GDPR***



***Full form: General Data Protection Regulation***



***GDPR*** is a law made by ***EU(EUROPIAN UNION)*** that governs how personally identifiable information is collected, processed, and eventually deleted from a computer system.



**Personal data is defined broadly in GDPR:**



* Basic identity information like name, address, and ID numbers.
* Health and genetic data.
* Biometric data.
* Racial data.
* Political opinions.

Cloud (or Cloud Computing) is the delivery of computing services—like servers, storage, databases, networking, software, and more—over the Internet ("the cloud").



***\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_***



* **Cloud Computing**	Using computing resources via the internet instead of local machines.



* **Cloud Provider**	A company that offers cloud services (e.g., AWS, Azure, Google Cloud).



* **Data Center**		Physical location with servers that host cloud services.



* **Virtualization**	Creating virtual versions of hardware/software (e.g., virtual machines).

***Types of Cloud***



**1. Public Cloud**



Services offered to anyone over the internet.



Example: Gmail, Google Drive, AWS EC2.



**--------------------------------------------------------------------------------------------------------------------------------------------------------------------**



**2. Private Cloud**



Used by a single organization; more control and security.



Example: Company-hosted internal servers accessed via VPN.



**--------------------------------------------------------------------------------------------------------------------------------------------------------------------**





**3. Hybrid Cloud**



Combination of public and private cloud.



Example: A company stores sensitive data on a private cloud and runs apps on a public cloud.

* **In cloud computing, IaaS, PaaS, and SaaS are three core service models. Each represents a different level of cloud service abstraction:**



***1. IaaS (Infrastructure as a Service)***



You rent virtual computers and storage.



You install and manage your own software on them.



Example: Amazon EC2 — rent virtual servers to run your apps.



**--------------------------------------------------------------------------------------------------------------------------------------------------------------------**



***2. PaaS (Platform as a Service)***



You get a ready-made platform to build and run apps without managing servers.



Example: Google App Engine — develop and host your apps easily.



**--------------------------------------------------------------------------------------------------------------------------------------------------------------------**



***3. SaaS (Software as a Service)***



You use software over the internet without installing it.



Everything is managed for you.



Example: Gmail — access your email online anywhere.

***Most Important Questions***



1. What cloud deployment model (public, private, hybrid) best fits the business needs and security requirements?
2. How does moving to the cloud impact existing business processes and user experience?
3. What are the key security and compliance standards (e.g., GDPR, HIPAA) the cloud solution must meet?
4. What integrations are required between the cloud system and existing on-premise or third-party applications?
5. What are the vendor’s Service Level Agreements (SLAs) for uptime, support, and incident response?
6. How will cloud costs be tracked, budgeted, and optimized over time?
7. What is the project timeline and key milestones for cloud adoption or migration?
8. What disaster recovery and data backup strategies does the cloud provider offer?
9. How will change management be handled to ensure smooth user adoption and training?
10. What metrics and analytics will be used to measure cloud performance, ROI, and business impact?

***Module 1: Introduction to Azure SQL Database***

**What is Azure SQL Database?**



* A cloud-based relational database service.



* Runs Microsoft SQL Server engine in the cloud.



* It is a PaaS (Platform as a Service), sometimes also referred to as DBaaS (Database as a Service).



* Fully managed by Microsoft—no need to worry about infrastructure, patching, or backups.



**--------------------------------------------------------------------------------------------------------------------------------------------------------------------**



***Azure SQL Deployment Models***





**1. Single Database**	Isolated database with dedicated resources. Ideal for microservices or individual apps.



**2. Elastic Pool**		Multiple databases share resources (DTUs/vCores). Great for unpredictable or bursty workloads.



**3. Managed Instance**	Near full compatibility with on-prem SQL Server. Ideal for lift-and-shift migrations.



**--------------------------------------------------------------------------------------------------------------------------------------------------------------------**



**Purchasing Model**



***DTU Model (Database Transaction Unit) TYPES***



1. <i>Basic		For small databases with light workloads.</i>
2. <i>Standard		For average workloads, moderate performance.</i>
3. <i>Premium		For high-transaction or high-concurrency workloads with low latency ne eds.</i>





* Pre-configured bundle of CPU, memory, and I/O



* Simple pricing with fixed performance tiers (Basic, Standard, Premium)



* Ideal for small to medium apps or beginners



* No support for Azure Hybrid Benefit (license reuse)



**--------------------------------------------------------------------------------------------------------------------------------------------------------------------**



***vCore Model (Virtual Core) TYPES***



1. <i>General Purpose	Balanced compute and storage; cost-effective for most workloads.</i>
2. <i>Business Critical	High-performance for I/O-intensive workloads, with lower latency and local SSDs.</i>
3. <i>Hyperscale		Supports very large databases (up to 100 TB), fast backups, and scale-out reads.</i>





* Choose your own CPU, memory, and storage



* More flexible and transparent pricing



* Suitable for enterprise or high-performance workloads



* Supports Azure Hybrid Benefit for cost savings
